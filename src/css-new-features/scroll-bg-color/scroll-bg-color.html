Scroll-driven background color

Shift the page background through a full hue arc as you scroll — no JS, no scroll listeners, no requestAnimationFrame. Just @property and animation-timeline.

↓ Scroll down to see the color shift

hue 220 — top of page
hue 260 — 25%
hue 300 — 50%
hue 330 — 75%
hue 350 — bottom

Step 1 — register the variable

Plain CSS custom properties can't be interpolated — the browser doesn't know if --bg-hue is a number, a color, or a string. Declare its type with @property so the browser can smoothly tween between values. 

@property --bg-hue {
  syntax: "<number>";
  inherits: true;
  initial-value: 220;
}

Step 2 — define the keyframes

A normal @keyframes block — but instead of moving something, we just change the hue number. 

@keyframes shift-bg {
  0%   { --bg-hue: 220; }
  100% { --bg-hue: 350; }
}

Step 3 — bind to scroll

Apply the animation to body, then replace the time-based clock with animation-timeline: scroll(). Progress is now tied to scroll position — not seconds. 

body {
  background: hsl(var(--bg-hue) 45% 13%);
  animation: shift-bg linear;
  animation-timeline: scroll();
}

Requires Chrome 115+, Firefox 110+, Safari 18+. In older browsers the background stays at its initial color.
